POST /auth/token

Overview

Exchanges client credentials for a short-lived bearer token. Use this token in the Authorization header for all subsequent API requests.

Authentication required: No  |  Rate limit: 10 requests/minute per client

Request

POST /auth/token Content-Type: application/json { "client_id": "abc123", "client_secret": "s3cr3t_value", "grant_type": "client_credentials" }

Successful Response — 200 OK

{ "access_token": "eyJhbGciOiJSUzI1NiIsInR...", "token_type": "Bearer", "expires_in": 3600, // seconds "scope": "read write" }

Error Responses

// 401 Unauthorized — invalid credentials { "error": "invalid_client", "error_description": "Client authentication failed." } // 429 Too Many Requests { "error": "rate_limit_exceeded", "retry_after": 60 }

Notes

  • Tokens expire after 3600 seconds. Implement token refresh logic before expiry.
  • Store client_secret server-side only — never expose in client-side code.
  • All API requests must include Authorization: Bearer <access_token>.

Getting Started with Dashboard Filters

Version: 3.4  |  Audience: Analyst, Admin roles  |  Time to complete: ~3 minutes


Overview
Dashboard Filters let you narrow any data view by date range, team, or project. Once configured, filters persist across sessions, so your preferred view is always ready when you log in.


Prerequisites

  • Account with Dashboard access (Admin or Analyst role)
  • At least one active Project in your workspace

Step 1: Open the Filters Panel
From the main Dashboard, click the Filters icon (⊞) in the top-right toolbar. The Filters panel slides in from the right.


Step 2: Select a Date Range
Under Date Range, choose a preset (Last 7 days, Last 30 days, This quarter) or click Custom to define start and end dates.


Step 3: Apply and Save
Click Apply Filters. To save this filter as your default view, toggle Save as default before applying.


💡 Tip: Filters stack — you can combine Date Range + Team + Project for a highly targeted view.

Troubleshooting

Filters not persisting after log-out?
Ensure you toggled Save as default before applying. Guest accounts cannot save filter preferences.


No projects appearing in the Project filter?
You need at least Viewer access on a Project for it to appear. Contact your workspace Admin to request access.

Documentation Framework Design

A sample documentation framework covering content taxonomy, naming conventions, review cycles, and version control — the kind of foundational work I do as a documentation leader.

Context

This sample reflects the documentation architecture work I did at Komatsu Pakistan Soft — building the frameworks, standards, and processes that allow a documentation team to scale consistently.

Content Taxonomy

docs/ ├─ getting-started/ │ ├─ quickstart.md │ └─ installation.md ├─ api-reference/ │ ├─ authentication/ │ ├─ endpoints/ │ └─ error-codes.md ├─ user-guides/ │ ├─ features/ │ └─ integrations/ ├─ release-notes/ └─ compliance/ iso-9001/ iso-27001/

Naming Conventions

  • All files: kebab-case, no spaces or special characters
  • API endpoint docs: verb-resource.md (e.g. create-user.md)
  • Versioned docs: suffix with -v{major}.md when breaking changes occur
  • Draft files: prefix with _draft- until approved

Review Cycle (Sprint-aligned)

Week 1: Writer drafts based on feature spec + engineering walkthrough

Week 2: SME technical review (engineering lead)

Week 3: Editorial review (Documentation Manager) + UX accuracy check

Week 4: Final approval + publish, aligned with sprint release
